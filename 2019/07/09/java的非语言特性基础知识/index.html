java 基础知识

Java非语言特性基础知识总结

1. Java的标识符

a. 标识符仅能由英文字母，数字，$，下划线构成，其中数字不可作为开头首字符

b. 标识符严格区分大小写

c. 不可使用语言的关键字

2. Java基础数据类型

a. byte

  • 8位，有符号的整数
  • -2^7~2^7-1（-128~127）
  • 默认值 0

b. short

  • 16位，有符号的整数
  • -2^15~2^15-1（-32,768~32,767）
  • 默认值 0

c. int

  • 32位。有符号的整数
  • -2^31~2^31-1（-2,147,483,648~2147483647）
  • 整数默认为int类型
  • 默认值 0

d. long

  • 64位，有符号的整数
  • -2^63~2^63-1
  • 在long定义整数后需要加上l/L(一般使用大写L，防止混淆),例：long x=1123L;
  • 默认值 0L

e. float

  • 单精度，32位，符合IEEE754标准的浮点数
  • 在float定义浮点数后需要加上f/F,例：float x=112.3f;
  • 默认值0.0f

f. double

  • 双精度，64位，符合IEEE754标准的浮点数
  • 浮点数默认类型位double
  • 默认值0.0d

g. boolean

  • 仅有两个取值true和false
  • 默认为false

h. char

  • 表示一个单一的16位Unicode字符
  • \u0000~\uffff（0~65535）

3. Java声明常量

a. 使用fianl，并且通常和static一起使用,例：public static final int PI=3.14;

b， 一般习惯使用大写字母表示常量

4. Java局部变量

a. 局部变量声明在方法，语句块中

b. 访问修饰符不可作用于局部变量

c. 尽在其所在的方法或语句块中可见

d. 局部变量没有默认值，被声明后必须初始化才可使用

5. Java实例变量

a. 声明在一个类之内，但是不在方法，构造方法，语句块中

b. 当一个对象实例化后，每个实例变量的值就跟着确定

c. 在对象创建时创建，在对象销毁时也一同销毁

d. 可以用访问修饰符修饰

e. 有默认值

f. 一般设为private,用set,get封装

6. Java静态变量

a. 类中定义以static关键字，且不能在方法以内，例： public static int num=1000;

b. 其他类访问时，直接（类名.属性名）即可调用

c. 通常使用public，通常使用于声明常量

7. Java访问修饰符

a. default就是什么都不写，在同一个包内可见

b. private在同一个类内可见

c. public 对所有类可见

d. protected 对同一个包内的类和所有子类(可以不再同一个包内)可见

*private和protected不可修饰类!

8. Java非访问修饰符

a. static可以声明静态变量和静态方法，静态方法不可使用类的非静态变量

b. final可以声明常量，方法和类，方法可以被继承但不可修改，类不可被继承，为最终类

c. abstract声明抽象类和抽象方法

  • 抽象类的目的就是给出一个模板为了将来对该类进行扩充
  • 一个类不可同时被abstract和final同时修饰，抽象类就是要为了将来被继承
  • 抽象类中可以有非抽象方法，但有抽象方法，则该方法一定是抽象类
  • 继承抽象类的子类，要么实现父类的方法，要么也是一个抽象类，将不能实现的方法继承下来
  • 抽象方法不可声明为final和static
  • 抽象方法以分号结尾 例：public abstract go();

c. 位运算

|, 按位或

用来判断对象是否是特定类型

例：（A）instanceof(B)

14. Java 类

a. 类是对象的抽象概念

b. Java中通过new来创建一个新的对象，抽象类不可被实例化

c. 类中要有构造方法，如果没有显式的声明一个构造方法，则Java编译器将会为该类提供一个默认构造方法

d. 类的继承extend，一个子类仅可继承一个父类，java不支持多重继承，

15. Java接口

a. 接口的定义interface ,接口可以看成百分之百的抽象类

b. 实现接口的定义implements 一个类可以实现多个接口，实现接口就要实现接口的所有犯法

c. 接口的继承，当然，也可以再写一个接口用extend继承其他接口，在进行功能的完善
