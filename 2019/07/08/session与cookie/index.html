session与cookie

发表于 2019-07-08 | 分类于 java web
字数统计: 448 | 阅读时长 ≈ 1

##session与cookie粗略介绍
session存在于服务端，cookie对应于相应的session存在于客户端，cookie（JSESSIONID）–session(SESSIONID)，服务端在相应请求时产生session，再对应生成cookie传送给客户端

##请求过程

###第一次请求：

此请求没有JSESSIONID，服务端产生一个session对象,包括用户的账号密码等各种信息，并且生成一个SESSIONID
然后产生一个cookie对象，结构为

cookie（name=JSESSSIONID,value=SESSIONID）

注：JESSIONID=SESSIONID，JSESSION是tomcat对SESSIONID的别称

把cookie发送给客户端，这样客户端就有了一个cookie(JSESSIONID)

服务端准备cookie:

response.addCookie(Cookie cookie)

##第二次请求：
客户端获取cookie:

request.getCookies();

拿着cookie去找，如果JSESSIONID=SESSIONID则匹配到

##session
session:

a. session存储在服务端

b. session是在 同一个用户（客户，浏览器）请求时 共享

c. 实现机制：第一次客户请求时 产生一个sessionid 并复制给 cookie的jsessionid 然后发给客户端。最终 通过session的sessionid-cookie的jsessionid

###session方法：

String getId() :获取sessionId

boolean isNew() :判断是否是 新用户（第一次访问）

void invalidate():使session失效 （退出登录、注销）

void setAttribute();

Object getAttribute();

void setMaxInactiveInterval(秒) ：设置最大有效 非活动时间

int getMaxInactiveInterval():获取最大有效 非活动时间

##Cookie：
a.不是内置对象，要使用必须new

b.但是，如果没有定义（new）一个返回的cookie,服务端会自动生成一个(服务端自动new一个cookie) name=JSESIONID的cookie,但不会new别的cookie 并返回给客户端

###cookie方法
public Cookie(String name,String value)

String getName()：获取name

String getValue():获取value

void setMaxAge(int expiry);最大有效期 （秒）

response.addCookie(Cookie cookie)

request.getCookies();

##session与cookie的区别
session:内置对象（不需要new），服务端，较安全，Object

cookie:非内置对象（需要new），客户端，较不安全，String
