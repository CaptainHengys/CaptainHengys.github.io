JSP九大内置对象中的四种范围对象

发表于 2019-07-08 | 分类于 java web
字数统计: 439 | 阅读时长 ≈ 1

##JSP九大内置对象

###常用的五种
1.pageContext JSP页面容器

2.request 请求对象

3.session 会话对象

4.appliation 全局对象

5.response 响应对象

###下面的四种不是很常用
6.config 配置对象（服务器配置信息）

7.out 输出对象

8.page 当前JSP页面对象（相当于java中的this）

9.exception 异常对象

##四种范围对象
其中前四种就是范围对象，范围从小到大依次为

###1.pageContext
JSP页面容器（书上经常会叫他page对象，平时提到的page那么一般就是指pageContext），仅在当前页面有效

####其他页面无法访问当前页面信息

###2.request
请求对象，仅在一次请求中有效

####什么叫同一次请求
请求的转移有两种，一种是请求转发，一种是重定向，请求转发仍然属于同一次请求，他的操作都在用户的同一次请求周期内，虽然思维想象中，像是当前页面发起新的请求，如果是重定向，是服务器让用户重新发出一个新的请求，显然重定向不属于同一次请求

###3.session
会话对象，尽在同一次会话中有效

####什么叫同一次会话
简单理解为，无论怎么跳转，怎么请求都有效，但是，如果关闭或者切换浏览器，或者进行清空session的注销操作等，session会话即刻无效

###4.application
全局对象，全局有效，整个项目有效

####整个项目运行期间都有效（无论切换网页，开关浏览器），但是服务器项目关闭当前项目，或者其他项目访问当前项目无效
